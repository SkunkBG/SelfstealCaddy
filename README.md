# SelfSteal Caddy Stub

Скрипт автоматической установки **SelfSteal**-заглушки для нод [Remnawave](https://github.com/remnawave) + Xray Reality.

Устанавливает [Caddy](https://caddyserver.com/) как локальный HTTPS-бэкенд с правдоподобным многостраничным сайтом, чтобы ваша Reality-нода маскировалась под **собственный домен** с валидным TLS-сертификатом — и, что не менее важно, вела себя как обычный статический сайт при активном зондировании TSPU/DPI.

## Как это работает

```
Клиент (SNI: your-domain.com)
        │
        ▼
  Xray Reality (:443)            ◄── публичная поверхность, выглядит как обычный HTTPS
        │
        ├── VPN-клиент? ──► VPN-туннель
        │
        └── Зонд/сканер? ──► проксирует на 127.0.0.1:8443
                                        │
                                        ▼
                               Caddy (валидный TLS-сертификат
                               для your-domain.com)
                                        │
                                        ▼
                              Многостраничный сайт-заглушка
```

DPI или активное зондирование видит легитимный сертификат, совпадающий с SNI, и обычный сайт малого бизнеса — ничего подозрительного.

## Что делает заглушку похожей на настоящий сайт

Смысл SelfSteal-страницы — быть неотличимой от обычного статического HTTPS-сайта при зондировании. Эта версия закрывает типовые маркеры self-steal:

- **Четыре разных типа сайта** — студия, кофейня, юрфирма, локальный подрядчик. Это не один шаблон с разным CSS: у каждого типа **своя структура DOM**, свой набор страниц и своя вёрстка. Поэтому ноды не склеиваются в единый структурный отпечаток.
- **Рандомизация по нодам** — при установке случайно выбираются тема, название бренда (из пула), акцентный цвет, город и год основания. На 20+ нодах это даёт 20+ разных сайтов вместо одного клонированного `index.html` на всех IP — а именно одинаковый контент на разных адресах выдаёт всю сеть разом.
- **Реальная глубина сайта** — несколько настоящих подстраниц (`/work.html`, `/menu.html`, `/practice.html` и т.п.), а не якоря `#section` на одной странице. Краулер-зонд, идущий по ссылкам, попадает на реальные документы.
- **Честные статус-коды** — `GET /` → `200`, несуществующие пути → настоящий `404`. Никакого SPA-фоллбэка с `200` на любой путь (классический маркер заглушек).
- **Сайт выглядит законченным** — `favicon.svg` (монограмма бренда) + `favicon.ico`, `robots.txt`, `sitemap.xml` со списком реальных страниц и `/.well-known/security.txt`. Браузероподобный зонд не натыкается на голые 404 по стандартным путям.
- **HTTP/3 отключён, `Alt-Svc` вырезан** — Caddy по умолчанию анонсирует `Alt-Svc: h3=":8443"`, рекламируя несуществующий снаружи QUIC-сервис и **засвечивая внутренний порт 8443**. Здесь оставлены только HTTP/1.1 и HTTP/2, заголовок `Alt-Svc` удалён.
- **`admin off`** — admin-API Caddy (localhost:2019) выключён полностью, поверхность сервера минимальна.
- **Убран заголовок `Server: Caddy`** + добавлены HSTS, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, адекватные `Cache-Control` — как у обычного хостинга.
- **Сжатие `zstd`/`gzip`, `ETag`/`Last-Modified`** от файлового сервера, чистый редирект `http://домен` → `https://домен` (порт 443, без `:8443` в `Location`).
- **Ноль внешних запросов, ноль JS** — только системные шрифты и общий same-origin `/style.css`. Никаких сторонних доменов в сетевых «следах».

> **Важно.** Страница защищает только от *активного зондирования* бэкенда. Форму трафика, которую генерируют клиенты, определяет Xray Reality / настройки инбаунда, а не эта страница. См. раздел [Усиление вне страницы](#усиление-вне-страницы).

## Требования

- Сервер на Debian / Ubuntu
- Root-доступ
- Домен с DNS A-записью, указывающей на IP сервера
- Нода Xray Reality (Remnawave или standalone)

## Установка

**Одной командой (интерактивный выбор темы):**

```
bash <(curl -Ls https://raw.githubusercontent.com/SkunkBG/SelfstealCaddy/main/selfsteal-setup.sh)
```

**Без интерактива** (тема и домен через переменные окружения):

```
DOMAIN=your-domain.com STUB_THEME=random \
  bash <(curl -Ls https://raw.githubusercontent.com/SkunkBG/SelfstealCaddy/main/selfsteal-setup.sh)
```

`STUB_THEME` принимает: `studio` · `coffee` · `law` · `contractor` · `random` (по умолчанию — `random`).

## Темы

| Тема         | Тип сайта             | Страницы                                  |
| ------------ | --------------------- | ----------------------------------------- |
| `studio`     | Дизайн-студия         | `index` · `studio` · `work` · `contact`   |
| `coffee`     | Кофейня               | `index` · `menu` · `about` · `visit`      |
| `law`        | Юридическая фирма     | `index` · `practice` · `people` · `contact` |
| `contractor` | Строительный подрядчик| `index` · `services` · `projects` · `contact` |

Бренд, акцентный цвет, город и год основания выбираются случайно при каждой установке.

## Тестовый прогон без установки

Можно сгенерировать сайт и конфиг локально, без root, apt, Caddy и UFW — для предпросмотра:

```
DRY_RUN=1 DOMAIN=example.com STUB_THEME=law \
  WEBROOT=/tmp/site CADDYFILE=/tmp/Caddyfile \
  bash selfsteal-setup.sh
```

Файлы окажутся в `/tmp/site`, конфиг — в `/tmp/Caddyfile`. Откройте `index.html` в браузере для проверки.

## Что делает скрипт

1. Запрашивает домен (или берёт из `DOMAIN`) и проверяет DNS-резолв
2. Устанавливает Caddy (если ещё не установлен)
3. Генерирует многостраничный сайт в `/var/www/html/` с рандомизированным брендингом, `sitemap.xml`, `security.txt`, `robots.txt`, favicon
4. Настраивает Caddy:
   - порт `80` — получение сертификата Let's Encrypt + редирект HTTP → HTTPS
   - порт `8443` — HTTPS с валидным сертификатом, HTTP/1.1+2 (без h3), сжатие, security/cache-заголовки, честные 404, `admin off`
5. Настраивает UFW (открывает `80`, `443`, закрывает `8443`)
6. Валидирует конфиг, запускает Caddy и проверяет, что `/` → `200`, а неизвестный путь → `404`

## После установки

Обновите конфиг Xray / Remnawave-ноды — измените два поля в `realitySettings`:

```
"realitySettings": {
    "target": "127.0.0.1:8443",
    "serverNames": ["your-domain.com"]
}
```

| Было                             | Стало                                |
| -------------------------------- | ------------------------------------ |
| `"target": "www.google.com:443"` | `"target": "127.0.0.1:8443"`         |
| `"serverNames": ["google.com"]`  | `"serverNames": ["your-domain.com"]` |

Всё остальное в конфиге (`shortIds`, `privateKey`, routing, outbounds) остаётся без изменений.

## Порты

| Порт   | Сервис       | Доступ                                        |
| ------ | ------------ | --------------------------------------------- |
| `443`  | Xray Reality | Публичный — VPN-подключения                   |
| `80`   | Caddy        | Публичный — обновление сертификата + редирект |
| `8443` | Caddy HTTPS  | **Только локальный** (`127.0.0.1`)            |

## Расположение файлов

| Файл              | Путь                                |
| ----------------- | ----------------------------------- |
| Страницы сайта    | `/var/www/html/*.html`              |
| Стили             | `/var/www/html/style.css`           |
| Карта сайта       | `/var/www/html/sitemap.xml`         |
| security.txt      | `/var/www/html/.well-known/security.txt` |
| robots.txt        | `/var/www/html/robots.txt`          |
| favicon           | `/var/www/html/favicon.svg` + `favicon.ico` |
| Конфиг Caddy      | `/etc/caddy/Caddyfile`              |
| Бэкап конфига     | `/etc/caddy/Caddyfile.bak.*`        |

## Усиление вне страницы

Заглушка закрывает зондирование бэкенда, но самые заметные маркеры часто лежат вне HTML:

- **Один домен на ноду, а не один домен на 30 IP.** A-запись «студии», резолвящаяся в десятки адресов у разных хостеров, — аномалия, видимая без всякого зондирования.
- **Дайте домену вид живого:** `www`, MX/SPF/DMARC TXT-записи, возраст, индексируемость. Свежий домен с одной страницей и без истории подозрителен сам по себе.
- **`nmap` по своему серверу снаружи.** Публично должны торчать только `80/443`. Панель Remnawave, management-порты, SSH на видном порту — куда более явные маркеры, чем любые заголовки страницы.
- **Не блокируйте и не меняйте ответы «подозрительным» сканерам.** Любая разница в поведении для разных клиентов (fail2ban на пробы, другой контент по User-Agent) — это и есть то, что ищет активное зондирование. Заглушка должна отвечать всем одинаково.
- **Сторона Reality:** `dest`==`serverNames`==свой домен, `maxTimeDiff: 0`. Несколько разных `shortIds` и вариативность клиентского uTLS-отпечатка (chrome/firefox/randomized) помогают против клиентской детекции.

## Кастомизация

Замените или отредактируйте любую страницу:

```
nano /var/www/html/index.html
systemctl restart caddy
```

Чтобы поменять логику генерации (пулы брендов, цветов, городов, тексты тем), правьте функции `gen_studio` / `gen_coffee` / `gen_law` / `gen_contractor` в `selfsteal-setup.sh`.

> **Внимание:** из-за `admin off` команда `systemctl reload caddy` работать не будет (reload использует admin-API). Применяйте изменения через `systemctl restart caddy`.

## Удаление

```
systemctl stop caddy
systemctl disable caddy
apt remove caddy -y
rm -rf /var/www/html
rm -f /etc/apt/sources.list.d/caddy-stable.list
rm -f /usr/share/keyrings/caddy-stable-archive-keyring.gpg
```

## Решение проблем

**Caddy не запускается:**

```
journalctl -u caddy --no-pager -n 30
```

**Сертификат не выдаётся:**

- Убедитесь, что порт `80` открыт и DNS A-запись указывает на сервер
- Проверьте: `curl -I http://your-domain.com`

**Проверка HTTPS вручную (локальный бэкенд):**

```
curl -I https://your-domain.com:8443
curl -o /dev/null -w "%{http_code}\n" https://your-domain.com:8443/nope   # ожидаем 404
```

**Проверка, что HTTP/3 не анонсируется:**

```
curl -sI https://your-domain.com:8443 | grep -i alt-svc   # должно быть пусто
```
